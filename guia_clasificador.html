¿Qué vamos a construir?

Un sistema que lee el texto de un correo universitario y automáticamente lo clasifica en una de 7 categorías útiles. Usamos TF-IDF + Regresión Logística, el combo clásico que funciona muy bien para clasificación de texto.

Las 7 Categorías

📚 tarea
Asignaciones, trabajos y entregas académicas
📝 nota
Calificaciones y resultados de exámenes
📢 anuncio
Avisos institucionales y comunicados
🗓️ evento
Conferencias, talleres y actividades
🏆 logro
Reconocimientos y premios académicos
✈️ intercambio
Movilidad, becas y convocatorias internacionales
⭐ PRIORIDAD ALTA

Información general no clasificada
💡 ¿Por qué Regresión Logística? Es el modelo ideal para texto + TF-IDF: rápido, interpretable, no necesita GPU, y alcanza ~85-95% de precisión con suficientes datos. Mucho mejor que Random Forest para texto.

Pipeline Completo de ML

1
Limpieza de Texto

Antes de entrenar, normalizamos el texto para que palabras como "Intercambio", "INTERCAMBIO" e "intercambio" sean la misma para el modelo.

import re

def limpiar_texto(texto: str) -> str:
    # 1. Minúsculas
    texto = texto.lower()
    
    # 2. Eliminar URLs y emails
    texto = re.sub(r'http\S+|www\S+', '', texto)
    texto = re.sub(r'\S+@\S+', '', texto)
    
    # 3. Conservar letras (incluyendo á,é,í,ó,ú,ñ)
    texto = re.sub(r'[^a-záéíóúñü\s]', '', texto)
    
    # 4. Espacios extra
    texto = re.sub(r'\s+', ' ', texto).strip()
    
    return texto

df["texto_limpio"] = df["texto"].apply(limpiar_texto)

Separar Datos

3
TF-IDF — Vectorización

Convierte texto en números que el modelo puede procesar. TF-IDF premia palabras específicas como "intercambio" o "convocatoria" y penaliza palabras genéricas.

TfidfVectorizer(
    ngram_range=(1, 2),  # "intercambio" Y "intercambio académico"
    max_features=5000,  # máximo 5000 características
    sublinear_tf=True,  # escala log, evita que docs largos dominen
    min_df=1,           # cada palabra cuenta (dataset pequeño)
)
💡 ¿Qué son los n-gramas? Con ngram_range=(1,2) el modelo aprende que "beca internacional" es diferente de "beca" sola. Esto ayuda mucho con la categoría intercambio.
4
Pipeline: Todo Junto
from sklearn.pipeline import Pipeline
from sklearn.linear_model import LogisticRegression

pipeline = Pipeline([
    ("tfidf", TfidfVectorizer(
        ngram_range=(1, 2),
        max_features=5000,
        sublinear_tf=True,
    )),
    ("modelo", LogisticRegression(
        max_iter=1000,
        class_weight="balanced",  # ← maneja desbalance
        random_state=42
    ))
])

pipeline.fit(X_train, y_train)     # entrenar
y_pred = pipeline.predict(X_test)  # predecir

Cómo Mejorar la Precisión

🎯 Problema: Confusión entre Categorías Similares

Las categorías que más se confunden son:

  • anuncio vs evento: ambos son comunicados. Solución: ejemplos donde el evento tiene fecha concreta.
  • intercambio vs anuncio: una convocatoria puede parecer un anuncio. Solución: palabras clave + más datos de intercambio.
  • logro vs nota: ambos hablan de calificaciones. Solución: logros hablan de "primer lugar", "reconocimiento", "distinción".
🛡️ Regla de Palabras Clave para "Intercambio"

Dado que esta categoría es crítica, usa una capa de reglas adicional:

PALABRAS_INTERCAMBIO = [
    "intercambio", "movilidad", "erasmus",
    "fulbright", "daad", "beca internacional",
    "exterior", "doble titulación"
]

def clasificar_con_reglas(texto):
    texto_lower = texto.lower()
    
    # Primero verificar palabras clave críticas
    for palabra in PALABRAS_INTERCAMBIO:
        if palabra in texto_lower:
            prob = pipeline.predict_proba([texto])[0]
            idx = list(pipeline.classes_).index("intercambio")
            if prob[idx] >= 0.15:  # umbral bajo = más sensible
                return "intercambio"
    
    # Si no, usar predicción normal del modelo
    return pipeline.predict([texto])[0]
📈 Estrategias de Mejora
1. Más datos
La mejora más impactante. Etiqueta manualmente correos reales de tu universidad.
2. class_weight='balanced'
Le dice al modelo que trate cada clase con igual importancia, aunque haya menos ejemplos.
3. Combinar asunto + cuerpo
texto = f"{asunto} {asunto} {cuerpo}" — el asunto se repite dos veces para darle más peso al clasificar.
4. Ajuste del umbral
Si la probabilidad máxima es < 60%, considera marcarlo como "revisar manualmente".
5. Data augmentation
Parafrasea ejemplos existentes con sinónimos para multiplicar los datos de entrenamiento.
6. Grid Search
Usa GridSearchCV para encontrar el mejor valor de C en LogisticRegression.
